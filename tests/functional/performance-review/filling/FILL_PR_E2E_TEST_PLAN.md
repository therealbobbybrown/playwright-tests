# План E2E теста Performance Review с выбором известных коллег

## Цель теста

Проверить полный цикл Performance Review с выбором конкретных коллег и заполнением анкет всеми участниками.

## Строгая последовательность шагов

### 1. Подготовка: Получить список активных пользователей

- Выполнить API запрос `OrgStructureHelper.getUsersList(3)`
- Получить минимум 3 активных пользователя с email и паролями
- Первый пользователь = админ (оцениваемый)
- Остальные пользователи = коллеги для выбора

### 2. Админ: Настройка Performance Review

- Залогиниться под админом (первый пользователь из списка)
- Перейти в настройки PR
- Включить все направления оценки:
  - Самооценка (self) - включена
  - Оценка от руководителя (manager) - включена
  - Оценка от подчиненных (subordinates) - включена
  - Оценка от коллег (colleagues) - включена
- Настроить подбор коллег:
  - Минимум: 1
  - Максимум: 2
  - Проверка руководителем: НЕТ
  - Ранний доступ: НЕТ

### 3. Админ: Создание Performance Review

- Перейти на страницу создания PR
- Выбрать первого пользователя (самого себя) как оцениваемого
- Добавить анкеты для ВСЕХ 4 направлений:
  1. Самооценка
  2. Оценка от руководителя
  3. Оценка от подчиненных
  4. Оценка от коллег
- Отправить на этап выбора коллег
- Получить ID созданного PR

### 4. Админ: Выбор коллег через UI

**ВАЖНО**: Не пропускать этот шаг! Коллеги выбираются через модальное окно.

- Отправить PR на этап подбора респондентов
- Открыть уведомления (иконка колокольчика)
- Кликнуть на уведомление "Перейти к оценке" (ПЕРВОЕ/ВЕРХНЕЕ уведомление)
- Должна открыться страница "Выбор коллег"
- Найти кнопку "Выбрать" и кликнуть
- Дождаться открытия модального окна со списком пользователей
- **Промотать модальное окно вниз до конца** (lazy loading данных)
- Промотать обратно вверх
- Выбрать 2 доступных коллег:
  - Проверять каждую строку на disabled (серый фон)
  - Если disabled - пропускать
  - Если активна - **кликать на всю строку** (не на чекбокс)
  - Повторить пока не выбрано 2 коллеги
- Нажать кнопку "Применить"
- Нажать кнопку "Отправить"
- Подтвердить отправку в модальном окне

### 5. Админ: Завершение этапа подбора

- Нажать кнопку "Завершить этап"
- Подтвердить завершение в модальном окне
- Нажать кнопку "Отправить анкеты"
- Подтвердить отправку анкет в модальном окне

### 6. Админ: Заполнение самооценки

- Открыть уведомления (иконка колокольчика)
- Найти уведомление про Performance Review (ПЕРВОЕ/ВЕРХНЕЕ)
- Кликнуть на "Перейти к оценке"
- **Если открылась страница выбора коллег** - кликнуть "Выбрать" → "Применить"
- Должна открыться форма заполнения анкеты
- Заполнить все 10 вопросов (выбирать варианты по шкале)
- Нажать кнопку "Отправить"
- Подтвердить отправку

### 7. Коллега: Заполнение оценки на админа

- Создать новый browser context для коллеги
- Залогиниться под первым выбранным коллегой (из массива `colleagues`)
- Перейти на главную страницу `/ru/`
- Открыть уведомления (иконка колокольчика)
- Найти уведомление про Performance Review - **ПЕРВОЕ (верхнее), не последнее!**
- Кликнуть на "Перейти к оценке"
- **Коллега НЕ должен выбирать своих коллег** - должна сразу открыться форма анкеты
- Если не открылась - искать кнопку "Заполнить анкету" на странице
- Заполнить все 10 вопросов
- Нажать кнопку "Отправить"
- Подтвердить отправку

## Критически важные моменты

### ❌ Чего НЕЛЬЗЯ делать:

1. **НЕЛЬЗЯ** пропускать шаги если элемент не найден - тест должен упасть с ошибкой
2. **НЕЛЬЗЯ** выбирать случайных коллег - только из списка активных пользователей
3. **НЕЛЬЗЯ** логиниться под пользователями не из списка активных
4. **НЕЛЬЗЯ** искать чекбоксы `input[type="checkbox"]` - их нет в UI
5. **НЕЛЬЗЯ** забывать промотать модальное окно вниз (lazy loading)
6. **НЕЛЬЗЯ** брать `.last()` уведомление - брать `.first()` (самое свежее сверху)

### ✅ Что обязательно делать:

1. Использовать `waitFor()` вместо `isVisible()` для критичных элементов
2. Кликать на всю строку `.row`, а не на чекбокс
3. Проверять disabled состояние строк перед кликом
4. Промотать модальное окно вниз и обратно вверх
5. Логировать каждый шаг для отладки
6. Делать скриншоты при ошибках
7. Брать ПЕРВОЕ (верхнее) уведомление, не последнее

## Ожидаемый результат

- PR создан с ID
- Выбраны 2 конкретных коллеги из активных пользователей
- Этап подбора завершен
- Анкеты отправлены
- Админ заполнил самооценку
- Коллега заполнил оценку на админа
- Тест проходит успешно
