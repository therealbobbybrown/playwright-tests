# Тесты для создания и прохождения опросов с отделами и группами

## Описание

Набор тестов для проверки функциональности создания опросов с выбором отделов и групп, их публикации и прохождения от имени пользователей из разных отделов/групп.

## Файлы тестов

### `surveys-create-and-pass-with-departments-groups.spec.js`

Содержит 3 теста:

1. **Создание опроса с выбором отделов** - создаёт опрос, выбирает отделы, публикует и проходит от имени разных пользователей
2. **Создание опроса с выбором групп** - создаёт опрос, выбирает группы, публикует и проходит
3. **Создание опроса с отделами и группами для тепловой карты** - создаёт опрос с вопросами типа "Шкала", выбирает отделы и группы, получает ответы и проверяет тепловую карту

## Расширенный Page Object

### `SurveyPublicationSettingsPage`

Добавлены новые методы:

- `selectAudienceInternal()` - выбрать аудиторию "Внутренний опрос"
- `assertParticipantsBlockVisible()` - проверить видимость блока "Кто участвует в опросе"
- `selectDepartments(departmentNames)` - выбрать отделы для опроса
- `selectGroups(groupNames)` - выбрать группы для опроса
- `selectDepartmentsAndGroups(departmentNames, groupNames)` - выбрать отделы и группы одновременно

## Особенности реализации

1. **Модальное окно UserQuerySelect**: Используется компонент `UserQuerySelect` с модальным окном `SheetModal`, которое содержит вкладки для отделов, групп и пользователей.

2. **Множественный выбор**: Компонент поддерживает множественный выбор отделов и групп.

3. **Прохождение от разных пользователей**: Тесты создают отдельные контексты браузера для каждого пользователя, используя разные `storageState` или логин через `LoginPage`.

4. **Тепловая карта**: Для отображения тепловой карты требуется:
   - Минимум 3 уникальных ответа от разных пользователей
   - Вопросы типа "Шкала" (с опцией "Использовать для построения тепловой карты")
   - Наличие отделов/групп в структуре компании

## Вспомогательные классы

### `OrgStructureHelper`

Класс для получения списков отделов и групп из орг структуры и управления ими:

**Получение списков:**

- `getDepartmentsList()` - получить список отделов из страницы "Настройка отделов"
- `getGroupsListFromPage()` - получить список групп из страницы управления группами
- `getGroupsList(cleanup)` - получить список групп через UserQuerySelect (создаёт временный опрос)
- `getDepartmentsListFromUserQuerySelect()` - получить список отделов через UserQuerySelect
- `getUsersList(limit)` - получить список пользователей из списка сотрудников

**Управление отделами:**

- `createDepartment(departmentName)` - создать отдел
- `deleteDepartment(departmentName)` - удалить отдел
- `addUsersToDepartment(departmentName, count)` - добавить пользователей в отдел
- `setDepartmentUsersCount(departmentName, targetCount)` - установить количество пользователей в отделе

**Управление группами:**

- `createGroup(groupName)` - создать группу пользователей
- `deleteGroup(groupName)` - удалить группу
- `addUsersToGroup(groupName, count)` - добавить пользователей в группу
- `setGroupUsersCount(groupName, targetCount)` - установить количество пользователей в группе

**Очистка:**

- `cleanup()` - удалить все созданные отделы и группы

### `StructureUserGroupsPage`

Page Object для работы со страницей управления группами пользователей:

- `openFromSideMenu()` - открыть страницу через боковое меню
- `createGroup(groupName)` - создать группу
- `renameGroup(newName)` - переименовать группу
- `openGroupByName(groupName)` - открыть группу по названию
- `addUsersToGroup(count)` - добавить пользователей в открытую группу
- `deleteGroup()` - удалить открытую группу
- `getGroupsList()` - получить список всех групп
- `getUsersCountInGroup()` - получить количество пользователей в открытой группе

**Особенности:**

- Методы автоматически открывают нужные страницы
- Списки получаются динамически для каждого стенда
- Метод `getGroupsList` создаёт временный опрос и может его удалить (параметр `cleanup`)

## Важные замечания

1. **Динамические данные**: Тесты автоматически получают списки отделов и групп из системы:
   - Отделы получаются из страницы "Настройка отделов" (дерево отделов)
   - Группы получаются из страницы управления группами или через UserQuerySelect
   - Списки будут разными для разных стендов (окружений)

2. **Автоматическое создание**: Если отделов/групп недостаточно для теста, они создаются автоматически:
   - Тесты создают отделы и группы при необходимости
   - Можно настроить количество пользователей в отделах/группах для разных кейсов
   - Созданные сущности можно удалить через метод `cleanup()`

3. **Управление составом**: Можно управлять количеством пользователей в отделах и группах:
   - `setDepartmentUsersCount()` - установить количество пользователей в отделе
   - `setGroupUsersCount()` - установить количество пользователей в группе
   - Это полезно для тестов анонимности (нужно минимум 3 пользователя)

4. **Минимум ответов**: Для отображения отдела/группы в тепловой карте требуется минимум 3 уникальных ответа от разных пользователей этого отдела/группы.

5. **Анонимность**: Отделы/группы с менее чем 3 ответами будут скрыты в тепловой карте согласно правилам анонимности.

6. **Контексты браузера**: Каждое прохождение опроса выполняется в отдельном контексте браузера, чтобы избежать конфликтов сессий.

## Примеры использования

### Базовое использование

```javascript
// Выбрать аудиторию "Внутренний опрос"
await publicationSettingsPage.selectAudienceInternal();

// Выбрать отделы
await publicationSettingsPage.selectDepartments([
  "Отдел разработки",
  "Отдел тестирования",
]);

// Выбрать группы
await publicationSettingsPage.selectGroups(["Группа разработки"]);

// Или выбрать и отделы, и группы одновременно
await publicationSettingsPage.selectDepartmentsAndGroups(
  ["Отдел разработки"],
  ["Группа тестирования"],
);
```

### Создание отделов и групп для тестов

```javascript
const orgHelper = new OrgStructureHelper(page, testInfo);

// Получить список отделов
const departments = await orgHelper.getDepartmentsList();

// Если отделов недостаточно, создать новые
if (departments.length < 2) {
  await orgHelper.createDepartment("Тест отдел 1");
  await orgHelper.createDepartment("Тест отдел 2");
}

// Установить количество пользователей в отделе (для тестов анонимности)
await orgHelper.setDepartmentUsersCount("Тест отдел 1", 3);

// Создать группу и добавить пользователей
await orgHelper.createGroup("Тест группа");
await orgHelper.setGroupUsersCount("Тест группа", 3);

// Очистить созданные сущности после теста
await orgHelper.cleanup();
```

### Получение списков для разных кейсов

```javascript
const orgHelper = new OrgStructureHelper(page, testInfo);

// Кейс 1: Отделы есть
const departments = await orgHelper.getDepartmentsList();
if (departments.length > 0) {
  // Используем существующие отделы
}

// Кейс 2: Отделов нет (пустая структура)
if (departments.length === 0) {
  // Тест для случая без отделов
}

// Кейс 3: Создать отделы с нужным количеством пользователей
await orgHelper.createDepartment("Отдел с 3 пользователями");
await orgHelper.setDepartmentUsersCount("Отдел с 3 пользователями", 3);

await orgHelper.createDepartment("Отдел с 2 пользователями");
await orgHelper.setDepartmentUsersCount("Отдел с 2 пользователями", 2);
```
