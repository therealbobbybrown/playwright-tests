# Тесты для функциональности опросов — Результаты

## Описание

Набор сложных тестов для проверки функциональности результатов опросов, включая:

- Пороги анонимности
- Баннеры и UI
- Структуру и категории
- Экспорт данных
- Фильтры и специальные случаи

## Файлы тестов

### 1. `surveys-results-anonymity-thresholds.spec.js`

Тесты для порогов анонимности (кейсы 46023, 46054, 46082, 46113, 46143):

- Проверка отображения/скрытия отделов в зависимости от количества уникальных ответов
- Проверка работы с константами БД
- Проверка системного минимума (3 ответа)

### 2. `surveys-results-banners-ui.spec.js`

Тесты для баннеров и UI (кейсы 46024, 46055, 46083, 46114):

- Проверка отображения/скрытия баннера анонимности
- Проверка динамического обновления баннера при применении фильтров
- Проверка скрытия тепловой карты при малом количестве ответов

### 3. `surveys-results-structure-categories.spec.js`

Тесты для структуры и категорий (кейсы 46025, 46056, 46084, 46115):

- Проверка категории "Без отдела"
- Проверка обратной совместимости (старые названия)
- Проверка работы с пустой структурой

### 4. `surveys-results-export.spec.js`

Тесты для экспорта (кейсы 46026, 46057, 46085, 46116):

- Проверка экспорта сырых данных (XLSX/CSV) с анонимностью
- Сравнение экспортированной тепловой карты с веб-версией
- Проверка наличия/отсутствия сносок в экспорте

### 5. `surveys-results-filters-special-cases.spec.js`

Тесты для фильтров и специальных случаев (кейсы 46027, 46058, 46086, 46117):

- Проверка логики "ИЛИ" в фильтрах
- Проверка отсутствия тепловой карты на публичной странице
- Проверка обратной совместимости для исторических данных
- Проверка работы со специальными опросами (ВиЛ1, ВиЛ2)

## Расширенный Page Object

Класс `SurveyResultsPage` был расширен следующими методами:

### Тепловая карта

- `assertHeatMapVisible()` — проверить видимость тепловой карты
- `assertHeatMapHidden()` — проверить скрытие тепловой карты
- `getVisibleDepartments()` — получить список видимых отделов
- `assertDepartmentVisible(departmentName)` — проверить видимость отдела
- `assertDepartmentHidden(departmentName)` — проверить скрытие отдела

### Баннеры

- `assertAnonymityBannerVisible()` — проверить видимость баннера
- `assertAnonymityBannerHidden()` — проверить скрытие баннера
- `getAnonymityBannerText()` — получить текст баннера

### Фильтры

- `applyDepartmentFilter(departmentName)` — применить фильтр по отделу
- `applyGroupFilter(groupName)` — применить фильтр по группе
- `applyMultipleFilters(filters)` — применить несколько фильтров
- `clearFilters()` — сбросить все фильтры

### Экспорт

- `exportToXLSX()` — экспортировать в XLSX
- `exportToCSV()` — экспортировать в CSV
- `exportToPPTX()` — экспортировать в PPTX

### Структура

- `assertNoDepartmentRowVisible()` — проверить видимость строки "Без отдела"
- `assertNoDepartmentRowHidden()` — проверить скрытие строки "Без отдела"
- `assertGeneralResultsVisible()` — проверить наличие секции "Общие результаты"

## Важные замечания

1. **Тестовые данные**: Большинство тестов требуют предварительной настройки опросов с определенным количеством ответов. В текущей версии тестов используются заглушки (TODO), которые нужно заменить на реальные данные.

2. **Зависимости**: Для тестов экспорта требуется библиотека `xlsx`, которая уже установлена в проект.

3. **Таймауты**: Все тесты помечены как `test.slow()` и имеют увеличенные таймауты (180 секунд), так как они выполняют сложные операции.

4. **Селекторы**: Селекторы в `SurveyResultsPage` обновлены на основе реального фронтенда:
   - Используются CSS-модули (классы вида `HeatMapResults_section`, `Results_results`)
   - Тепловая карта находится в компоненте `HeatMapResults`
   - Фильтры используют `UserSelect` с id `filter-users`
   - Экспорт через кнопку "Скачать результаты" с выпадающим меню
   - Баннер анонимности ищется по тексту (может быть в разных местах)
