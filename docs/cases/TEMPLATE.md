# Шаблон текстового кейса

Заполни поля текстом — без кода. Один файл может содержать несколько кейсов (разделяй заголовками). Пиши так, чтобы сразу было понятно, что считать входом, что считать эталоном для сравнения, и как считать итог.

```
Название: [кратко, что проверяем]
Тип: UI | API
Приоритет: high | medium | low
Теги: выбери подходящее — например @ui @api @smoke @regression @auth @security @surveys @feedback @objectives @performance-review @org-structure @virtualcurrency @critical @high

Стенд: [номер/URL стенда]
Роль/данные:
- роль: [admin/user/manager/...]
- креды: [не вставляй логин/пароль; если не из .env, напиши откуда их взять, например `.env.st7`]
- доп. данные: [id, payload, сущности, источники данных/файлов для сверки]

Предпосылки:
- [что должно существовать до теста или что подготовить; явно укажи нужную страницу/логин, если тест начинается не с логина]

Шаги:
1) ...
2) ...
3) ...

ОР:
1) [результат шага 1: что должно быть видно/вернуться/измениться; входные данные, порядок/сортировка, критерий успеха]
2) [результат шага 2: ...]
3) [...]

Нюансы:
- [skip при отсутствии данных? особые таймауты? очистка стейта? требования к окружению/браузеру? включать ли архивных/уволенных сотрудников?]

Статус:
- черновик | в работе | готов (заполняем, чтобы понимать прогресс)
```
