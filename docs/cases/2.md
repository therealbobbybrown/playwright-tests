# Настройка виртуальной валюты. Сверка балансов и истории операций

Тип: UI | API  
Приоритет: medium  
Теги: @ui @api @virtualcurrency  

Стенд: https://client.st7.apprs.ru/  
Роль/данные:
- роль: admin
- креды: `.env.st7`
- источник отчёта: XLSX «Балансы сотрудников <companyName>»
- источник истории: https://client.st7.apprs.ru/ru/manager/karma/transactions/ (пагинация до последней страницы)
- (опционально) API/БД: если есть endpoint/доступ для операций/балансов, указать и использовать

Предпосылки:
- Админ залогинен, доступна страница настроек виртуальной валюты и история операций.
- Выгрузка XLSX с актуальным балансом сотрудников получена (см. кейс «Скачивание балансов сотрудников»).

Шаги:
1) Открыть историю операций: https://client.st7.apprs.ru/ru/manager/karma/transactions/.
2) Собрать все страницы истории (пагинация «вперёд/назад»), для каждой строки:
   - «Количество» разобрать с учётом знака и валюты (например, -300 или +5).
   - «Получатель» нормализовать: убрать префикс буквы/перенос строки; привести к формату «Фамилия Имя…» (если в таблице «Имя Фамилия», сменить порядок).
   - Накопить сумму по каждому получателю (с учётом знака).
3) В XLSX отчёте «Балансы сотрудников <companyName>» для каждого ФИО просуммировать «Валюта для трат в магазине» + «Валюта для благодарности».
4) Сопоставить списки: все ФИО из отчёта присутствуют в истории после нормализации имён; у каждого получателя сумма операций не пустая.
5) (Если определены правила расчёта баланса из истории: период, типы операций, стартовый остаток) — пересчитать историю по этим правилам и сверить суммы с отчётом.

ОР:
1) История открыта, все страницы пройдены, суммы по получателям посчитаны с нормализацией ФИО.
2) Отчёт (XLSX) разобран, суммы валют по каждому сотруднику посчитаны.
3) Перечень получателей совпадает между историей и отчётом; нет пропущенных/дублирующих имён после нормализации, суммы по получателям не пустые.
4) Если задано правило расчёта баланса из истории (период/типы операций/стартовый остаток) — суммы совпадают с отчётом.
5) Если правило не задано — зафиксировано, что сверка сумм не выполняется (проверено только наличие получателей и непустые суммы).

Нюансы:
- Без явного бизнес-правила вычисления баланса из истории (фильтр по дате/типам операций/стартовому остатку) совпадение сумм может не быть гарантировано; в этом случае проверяем наличие получателей и непустые суммы.
- «Получатель» в истории может содержать первую букву + перенос строки; отчёт — «Фамилия Имя…»; нормализация обязательна.
- Пагинация должна быть пройдена полностью (стрелки/кнопки «вперёд/назад»).
- Уточнить, нужно ли включать архивных/уволенных сотрудников в сверку.
